загрузка linux
	- bios
		шукає і завантажує MBR
	- mbr
		master boot record, розміщується в першому секторі загрузочного диска (менше 512 байт)
			- головна загрузочна інформація (446 байт)
			- інфомація про таблицю розділів (64 байт)
			- 2 байти - для перевірки коректності mbr
	- grub
		Grand Unified Bootloader, якщо встановлено більше 1 ядра, то є можливість вибрати яке ядро завантажувати (або завантажить те, що по замовчуванню)
		завантажує і виконує образи ядра і initrd - Initial RAM Disk(невеликий набір модулів ядра, який лежить в запакованому вигляді поруч з ядром, може містити різні драйвери(для LVM наприклад), викор. як 
		тимчасова коренева файлова система)
	- kernel
		монтує файлову систему, виконує /sbin/init
	- init
		дивиться в файл /etc/inittab, щоб визначити runlevel і завантажує всі необхідні програми (фонові процеси у вигляді демонів)
	- runlevel
		0 – прервать выполнение
		1 – Однопользовательский режим, так называемый «Single user mode», или иными словами, консоль восстановления
		2 – Многопользовательский режим без поддержки NFS
		3 – Полноценный многопользовательский режим
		4 – не используется
		5 – X11
		6 – перезагрузка

daemon
	фоновий процес, який працює не залежно від терміналу, в якому був запущений, майже завжди демонт створюються процесом init
	якщо створити процес-нащадок і зразу припинити його батька, то цей процес стане "сиротою", потім його батьком стане init - процес стане демоном

LDAP (Lightweight Directory Access Protocol)
	клієнт-верверний протокол для доступу до служби каталогів
	LDAP в основному використовується для зберігання облікових записів, дозволяє центролізувати керування користувачами, групами, ...,
	можна дозволити користувачам підключатись і проходити аутентифікаію на будь-якому компі, який входить в нашу локальну мережу (потрібно ставити сервер, і на кодну машину клієнт)
	
cgroups - механізм ядра, який обмежує і ізолює обчислювальні ресурси для груп процесів
https://habr.com/company/selectel/blog/303190/
	можливості:
		- обмеження ресурсів
		- пріорітезація (різним групам можна виділити різні об"єми ресурсів)
		- облік витрат ресурсів групою
		- ізоляція (розділення простору імен для груп таким чином, що одній групі недоступні процеси, мережеві з"єднання і файли другої)
		- керування (призупинення freezing груп, створення контрольної точки і перезавантаження)
	 ls /sys/fs/cgroup/ - вивести список підсистем
		
namespaces (простір імен) - механізм ядра, який забезпечує ізоляцію процесів один від одного
https://habr.com/company/selectel/blog/279281/
	6 типів неймспейсів:
		- PID - ізолює PID процесів
		- NETWORK - ізолює мережеві пристрої, стеки, порти ...
		- USER - ID користуваців і груп
		- MOUNT - точки монтування
		- IPC - SystemV IPC, черги повідомлень POSIX
		- UTS - ім"я хоста і доменне ім"я NIS
Cgroups = limits how much you can use;
namespaces = limits what you can see (and therefore use)

Зомбі процес - запис в таблиці процесів. Ядро зберігає PID і exit-статус мертвого процеса, поки батьківський процес не прочитає його, зомбі існує, доки живий його батьківськи процес
	або поки батьківський процес не запитає в ядра код завершення потомка

LVM Logical Volume Management - уравління логічними томами
https://habr.com/post/67283/
	дозволяє зібрати в один диск багато різних дисків, потім розбити їх так, як треба
		- PV - physical volumes
		- VG (Volume Group) - об"єднує PV в групу (один диск)
		- LV (Logical Volume) - логічні розділи, які робимо на нашому VG
